# 部署指南

## 关于数据库

**好消息：不需要手动注册数据库账号！**

本项目使用 **SQLite** 数据库，它是一个本地文件数据库，具有以下特点：

✅ **无需安装** - Python自带支持  
✅ **无需注册** - 不需要账号密码  
✅ **自动创建** - 首次运行自动创建数据库文件  
✅ **零配置** - 开箱即用  

数据库文件会自动创建在 `database/pcba_detection.db`，你不需要做任何额外操作。

## 快速开始

### 1. 安装依赖

```bash
pip install -r requirements.txt
```

### 2. 确保模型文件存在

确保 `models/best.pt` 文件存在（已经帮你复制了）

### 3. 运行项目

```bash
python run.py
```

### 4. 访问系统

打开浏览器访问：`http://localhost:5000`

## 部署到GitHub

### 前端部署（GitHub Pages）

1. 将 `frontend/` 目录的内容推送到GitHub
2. 在GitHub仓库设置中启用GitHub Pages
3. 选择 `frontend` 目录作为源

### 后端部署（推荐Railway）

1. 注册Railway账号：https://railway.app
2. 连接GitHub仓库
3. Railway会自动检测Flask项目
4. 设置环境变量（如果需要）
5. 部署完成！

## 数据库迁移（可选）

如果以后需要迁移到PostgreSQL（生产环境推荐）：

1. 修改 `backend/config.py` 中的数据库URI
2. 安装PostgreSQL驱动：`pip install psycopg2-binary`
3. 数据库会自动迁移（SQLAlchemy会处理）

## 常见问题

**Q: 数据库在哪里？**  
A: 在 `database/pcba_detection.db`，首次运行自动创建

**Q: 需要安装数据库软件吗？**  
A: 不需要，SQLite是文件数据库，Python自带支持

**Q: 如何备份数据库？**  
A: 直接复制 `database/pcba_detection.db` 文件即可

**Q: 如何查看数据库内容？**  
A: 可以使用SQLite浏览器工具，如DB Browser for SQLite
