# 🚀 完整部署步骤（详细版）

## 第一步：准备Git仓库

### 1.1 初始化Git（如果还没有）

```bash
# 在项目根目录执行
git init
```

### 1.2 检查.gitignore文件

确保 `.gitignore` 文件存在且包含：
- `venv/` - 虚拟环境
- `*.db` - 数据库文件
- `uploads/` - 上传文件（可选，如果太大）

## 第二步：创建GitHub仓库

### 2.1 在GitHub创建仓库

1. 访问：https://github.com/new
2. Repository name: `pcba-detection-system`
3. Description: `PCBA智能缺陷检测系统 - 基于YOLO的Web应用`
4. 选择 **Public**（GitHub Pages需要公开仓库）
5. **不要**勾选任何初始化选项
6. 点击 "Create repository"

### 2.2 上传代码到GitHub

在项目根目录执行：

```bash
# 添加所有文件
git add .

# 提交
git commit -m "Initial commit: PCBA检测系统完整项目"

# 添加远程仓库（替换YOUR_USERNAME）
git remote add origin https://github.com/YOUR_USERNAME/pcba-detection-system.git

# 推送到GitHub
git branch -M main
git push -u origin main
```

**如果遇到认证问题**：
- 使用GitHub Personal Access Token
- 或使用GitHub Desktop图形界面

## 第三步：部署前端到GitHub Pages

### 3.1 启用GitHub Pages

1. 进入你的GitHub仓库页面
2. 点击 **Settings**（设置）
3. 左侧菜单找到 **Pages**
4. 在 "Source" 部分：
   - 选择 "Deploy from a branch"
   - Branch: `main`
   - Folder: `/frontend` ⚠️ 重要：选择frontend文件夹
5. 点击 **Save**

### 3.2 等待部署完成

- GitHub会在几分钟内构建和部署
- 部署完成后，你会看到绿色提示
- 你的网站地址：`https://YOUR_USERNAME.github.io/pcba-detection-system/`

### 3.3 测试前端

访问你的GitHub Pages地址，应该能看到页面（但API调用会失败，因为后端还没部署）

## 第四步：部署后端到Railway

### 4.1 注册Railway账号

1. 访问：https://railway.app
2. 点击 "Login"
3. 选择 "Login with GitHub"（推荐，方便连接仓库）

### 4.2 创建新项目

1. 登录后，点击 **"New Project"**
2. 选择 **"Deploy from GitHub repo"**
3. 授权Railway访问你的GitHub账号（如果还没授权）
4. 选择你的仓库：`pcba-detection-system`
5. Railway会自动开始部署

### 4.3 配置环境变量（如果需要）

Railway会自动检测Python项目，通常不需要额外配置。

如果需要，可以在Railway项目设置中添加环境变量：
- `FLASK_ENV=production`
- `SECRET_KEY=your-secret-key`

### 4.4 获取后端URL

1. 部署完成后，Railway会给你一个URL
2. 格式类似：`https://pcba-detection-production.up.railway.app`
3. **复制这个URL**，下一步需要用到

### 4.5 设置自定义域名（可选）

Railway支持自定义域名，但免费版可能有限制。

## 第五步：连接前端和后端

### 5.1 修改前端API地址

1. 在GitHub仓库中，编辑文件：`frontend/js/api.js`
2. 找到这一行：
```javascript
const API_BASE_URL = 'https://YOUR_RAILWAY_APP.railway.app/api';
```
3. 替换 `YOUR_RAILWAY_APP` 为你的实际Railway地址
4. 例如：
```javascript
const API_BASE_URL = 'https://pcba-detection-production.up.railway.app/api';
```

### 5.2 提交更改

```bash
git add frontend/js/api.js
git commit -m "Update API URL to Railway backend"
git push
```

### 5.3 等待GitHub Pages更新

- GitHub Pages会自动重新部署
- 等待1-2分钟

## 第六步：测试完整系统

### 6.1 测试前端

访问：`https://YOUR_USERNAME.github.io/pcba-detection-system/`

### 6.2 测试功能

1. 上传一张PCBA图片
2. 点击"开始检测"
3. 查看检测结果
4. 查看历史记录

### 6.3 检查后端日志

在Railway项目页面，可以查看实时日志，确认请求是否正常。

## 🎉 完成！

现在你的系统已经部署完成，其他人可以通过以下地址访问：

**前端地址**：`https://YOUR_USERNAME.github.io/pcba-detection-system/`

## 📝 常见问题

### Q: GitHub Pages显示404？
A: 检查Settings → Pages中是否选择了正确的文件夹（/frontend）

### Q: API调用失败？
A: 检查：
1. Railway后端是否正常运行
2. API地址是否正确
3. CORS是否配置（已在代码中配置）

### Q: Railway部署失败？
A: 检查：
1. `requirements.txt` 是否存在
2. `run.py` 是否存在
3. 查看Railway日志了解具体错误

### Q: 如何更新代码？
A: 
1. 本地修改代码
2. `git add .`
3. `git commit -m "更新说明"`
4. `git push`
5. Railway和GitHub Pages会自动重新部署

## 🔒 安全提示

1. **不要**在代码中硬编码敏感信息
2. 使用环境变量存储密钥
3. 定期更新依赖包
4. 监控Railway使用量（免费额度有限）
