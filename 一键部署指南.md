# 🚀 一键部署到GitHub指南

## 📋 部署概览

你的项目将分为两部分部署：
- **前端** → GitHub Pages（免费，自动部署）
- **后端** → Railway（免费额度，自动部署）

最终效果：别人打开你的GitHub Pages网址就能使用完整系统！

---

## 第一步：安装Git（如果还没安装）

### Windows用户：
1. 下载：https://git-scm.com/download/win
2. 安装时全部选择默认选项
3. 安装完成后重启命令行

### 验证安装：
打开新的命令行窗口，输入：
```bash
git --version
```
如果显示版本号，说明安装成功。

---

## 第二步：创建GitHub仓库

### 2.1 登录GitHub
访问：https://github.com 并登录

### 2.2 创建新仓库
1. 点击右上角 **"+"** → **"New repository"**
2. 填写信息：
   - **Repository name**: `pcba-detection-system`
   - **Description**: `PCBA智能缺陷检测系统`
   - **选择 Public** ⚠️（GitHub Pages需要公开仓库）
   - **不要**勾选任何初始化选项
3. 点击 **"Create repository"**

### 2.3 复制仓库地址
创建后会显示仓库地址，类似：
```
https://github.com/YOUR_USERNAME/pcba-detection-system.git
```
**记住这个地址，下一步需要用到！**

---

## 第三步：上传代码到GitHub

### 方法A：使用命令行（推荐）

在项目根目录（`E:\挑战杯\代码\代码`）打开命令行，执行：

```bash
# 1. 初始化Git（如果还没初始化）
git init

# 2. 添加所有文件
git add .

# 3. 提交
git commit -m "Initial commit: PCBA检测系统"

# 4. 添加远程仓库（替换YOUR_USERNAME为你的GitHub用户名）
git remote add origin https://github.com/YOUR_USERNAME/pcba-detection-system.git

# 5. 推送到GitHub
git branch -M main
git push -u origin main
```

**如果提示需要登录**：
- 使用GitHub Personal Access Token（推荐）
- 或使用GitHub Desktop（见方法B）

### 方法B：使用GitHub Desktop（更简单）

1. **下载GitHub Desktop**：https://desktop.github.com
2. **安装并登录**你的GitHub账号
3. **添加仓库**：
   - File → Add Local Repository
   - 选择项目文件夹：`E:\挑战杯\代码\代码`
4. **发布到GitHub**：
   - 点击 "Publish repository"
   - 选择仓库名称和描述
   - 确保勾选 "Keep this code private" 是**未选中**状态
   - 点击 "Publish repository"

---

## 第四步：部署前端到GitHub Pages

### 4.1 启用GitHub Pages

1. 进入你的GitHub仓库页面
2. 点击 **"Settings"**（设置，在仓库顶部菜单）
3. 左侧菜单找到 **"Pages"**
4. 在 "Source" 部分：
   - 选择 **"Deploy from a branch"**
   - Branch: 选择 **`main`**
   - Folder: 选择 **`/frontend`** ⚠️ **重要！必须选择frontend文件夹**
5. 点击 **"Save"**

### 4.2 等待部署

- GitHub会在1-2分钟内构建和部署
- 部署完成后，你会看到绿色提示框
- 你的网站地址：`https://YOUR_USERNAME.github.io/pcba-detection-system/`

### 4.3 测试前端

访问你的GitHub Pages地址，应该能看到页面（但API会失败，因为后端还没部署）

---

## 第五步：部署后端到Railway

### 5.1 注册Railway账号

1. 访问：https://railway.app
2. 点击 **"Login"**
3. 选择 **"Login with GitHub"**（推荐，方便连接仓库）

### 5.2 创建新项目

1. 登录后，点击 **"New Project"**
2. 选择 **"Deploy from GitHub repo"**
3. 如果第一次使用，需要授权Railway访问GitHub
4. 选择你的仓库：`pcba-detection-system`
5. Railway会自动检测项目并开始部署

### 5.3 等待部署完成

- Railway会自动：
  - 检测Python项目
  - 安装依赖
  - 运行 `python run.py`
- 部署通常需要2-5分钟

### 5.4 获取后端URL

1. 部署完成后，在Railway项目页面
2. 点击你的服务
3. 在 "Settings" → "Domains" 可以看到URL
4. 格式类似：`https://pcba-detection-production.up.railway.app`
5. **复制这个完整URL**（包含https://）

---

## 第六步：连接前端和后端

### 6.1 修改前端API地址

1. 在GitHub仓库页面，点击 `frontend/js/api.js` 文件
2. 点击右上角 **"✏️ Edit"**（编辑按钮）
3. 找到这一行：
```javascript
const API_BASE_URL = 'https://YOUR_RAILWAY_APP.railway.app/api';
```
4. 替换为你的实际Railway地址，例如：
```javascript
const API_BASE_URL = 'https://pcba-detection-production.up.railway.app/api';
```
5. 滚动到底部，点击 **"Commit changes"**

### 6.2 等待GitHub Pages更新

- GitHub Pages会自动重新部署
- 等待1-2分钟

---

## 第七步：测试完整系统

### 7.1 访问你的网站

打开：`https://YOUR_USERNAME.github.io/pcba-detection-system/`

### 7.2 测试功能

1. 上传一张PCBA图片
2. 点击"开始检测"
3. 查看检测结果
4. 查看历史记录页面

### 7.3 如果遇到问题

**API调用失败？**
- 检查Railway后端是否正常运行（查看Railway日志）
- 检查API地址是否正确
- 检查浏览器控制台错误信息

**页面显示404？**
- 检查GitHub Pages设置中是否选择了 `/frontend` 文件夹
- 等待几分钟让GitHub完成部署

## ❓ 常见问题

### Q: Railway免费额度够用吗？
A: Railway提供$5/月免费额度，对于小型项目通常足够。可以随时查看使用量。

### Q: 可以自定义域名吗？
A: 可以！GitHub Pages和Railway都支持自定义域名。

### Q: 如何查看Railway日志？
A: 在Railway项目页面，点击你的服务，可以看到实时日志。

### Q: 数据库数据会丢失吗？
A: Railway免费版在项目休眠时可能会重置，建议定期备份数据库。
