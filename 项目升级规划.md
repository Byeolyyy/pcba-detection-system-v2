## 🎯 需要完成的环节

### 1. **后端API服务** (Python Flask/FastAPI)

#### 1.1 核心功能模块
- ✅ **检测服务** (`detection_service.py`)
  - 复用现有的YOLO检测逻辑
  - 处理图片/视频上传
  - 返回检测结果（JSON格式）

- ✅ **数据库模型** (`models.py`)
  - 检测记录表 (DetectionRecord)
    - id (主键)
    - defect_name (缺陷名称)
    - defect_code (缺陷编号)
    - confidence (置信度)
    - coordinates (坐标: x1, y1, x2, y2)
    - image_path (原图路径)
    - result_image_path (检测结果图路径)
    - created_at (创建时间)
    - user_id (用户ID，可选)

- ✅ **API路由** (`routes.py`)
  - `POST /api/detect` - 上传图片进行检测
  - `GET /api/records` - 获取所有检测记录
  - `GET /api/records/<id>` - 获取单条记录
  - `DELETE /api/records/<id>` - 删除记录
  - `GET /api/statistics` - 获取统计信息

#### 1.2 技术栈
- **框架**: Flask 或 FastAPI (推荐FastAPI，自动生成API文档)
- **数据库**: SQLite (开发) / PostgreSQL (生产)
- **ORM**: SQLAlchemy
- **文件存储**: 本地文件系统 或 云存储 (OSS/S3)

### 2. **前端Web界面** (HTML/CSS/JavaScript)

#### 2.1 页面结构
- ✅ **主页面** (`index.html`)
  - 图片上传区域
  - 实时预览
  - 检测结果展示
  - 检测历史列表

- ✅ **历史记录页面** (`history.html`)
  - 检测记录列表
  - 筛选和搜索
  - 详情查看
  - 统计图表

#### 2.2 技术栈
- **基础**: HTML5, CSS3, JavaScript (ES6+)
- **UI框架**: Bootstrap 5 或 Tailwind CSS
- **图表**: Chart.js (用于统计展示)
- **HTTP请求**: Fetch API 或 Axios

### 3. **数据库设计**

#### 3.1 表结构

```sql
-- 检测记录表
CREATE TABLE detection_records (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    defect_name VARCHAR(100) NOT NULL,      -- 缺陷名称
    defect_code VARCHAR(50),                -- 缺陷编号
    confidence DECIMAL(5,2) NOT NULL,       -- 置信度 (0-100)
    x1 INTEGER NOT NULL,                    -- 左上角x坐标
    y1 INTEGER NOT NULL,                    -- 左上角y坐标
    x2 INTEGER NOT NULL,                    -- 右下角x坐标
    y2 INTEGER NOT NULL,                    -- 右下角y坐标
    image_path VARCHAR(500),                -- 原图路径
    result_image_path VARCHAR(500),         -- 检测结果图路径
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 缺陷类型表 (可选)
CREATE TABLE defect_types (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    code VARCHAR(50) UNIQUE NOT NULL,       -- 缺陷编号
    name VARCHAR(100) NOT NULL,             -- 缺陷名称
    description TEXT,                       -- 缺陷描述
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```
