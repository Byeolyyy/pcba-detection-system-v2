# 彻底解决大文件问题

## 问题原因
`main_window.exe` 文件已经在Git提交历史中了，即使从工作区移除，历史记录中仍然存在。

## 解决方案：重置Git历史（最简单）

因为这是初始提交，还没有其他人使用，可以安全地重置历史。

### 步骤1：删除Git历史（保留文件）

在你的命令行执行：

```bash
# 删除.git目录（这会删除所有Git历史，但保留你的文件）
rmdir /s /q .git
```

**Windows PowerShell用户**：
```powershell
Remove-Item -Recurse -Force .git
```

### 步骤2：重新初始化Git

```bash
# 重新初始化
git init

# 配置用户信息（如果还没配置）
git config user.name "Byeolyyy"
git config user.email "your-email@example.com"
```

### 步骤3：确保.gitignore正确

检查 `.gitignore` 文件包含：
```
*.exe
main_window.exe
dist/
build/
```

### 步骤4：重新添加和提交

```bash
# 添加文件（.gitignore会排除大文件）
git add .

# 检查要提交的文件（确认没有exe文件）
git status

# 提交
git commit -m "Initial commit: PCBA检测系统"

# 添加远程仓库
git remote add origin https://github.com/Byeolyyy/pcba-detection-system.git

# 推送
git branch -M main
git push -u origin main
```

## 如果还是失败

如果推送时仍然提示大文件，检查是否有其他大文件：

```bash
# 查看所有大于50MB的文件
git ls-files | xargs ls -lh | awk '$5 ~ /M/ {if ($5+0 > 50) print $5, $9}'
```

或者手动检查：
```bash
# 查看要提交的文件大小
git ls-files -z | xargs -0 du -h | sort -rh | head -20
```

## 替代方案：使用Git LFS（如果必须保留exe）

如果你确实需要上传exe文件，使用Git LFS：

```bash
# 1. 安装Git LFS
# 下载：https://git-lfs.github.com/

# 2. 初始化
git lfs install

# 3. 跟踪exe文件
git lfs track "*.exe"
git lfs track "dist/**"

# 4. 添加.gitattributes
git add .gitattributes

# 5. 重新添加文件
git add .
git commit -m "Use Git LFS for large files"
git push -u origin main
```

## 推荐方案

**推荐使用重置历史的方法**，因为：
- exe文件不需要上传（用户可以从源码运行）
- 减少仓库大小
- 加快克隆速度
